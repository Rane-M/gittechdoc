Introduction

Git is a distributed version control system that helps developers track changes in their codebase. Unlike traditional version control systems, Git allows multiple developers to work on a project simultaneously without interfering with each other's work. It records changes, keeps a history of modifications, and makes it easy to collaborate and merge updates.

Created by Linus Torvalds in 2005, Git has become the most widely used version control system, supporting projects of all sizes. Whether you are a solo developer or part of a large team, Git enables efficient collaboration and code management.

Installation

Follow the instructions depending on which operating system you are using:

  • Windows: Download and install from git-scm.com
  • Linux: Use your package manager sudo apt install git (Debian/Ubuntu).
  • macOS: Install using Homebrew brew install git.

Basic Commands

Git Commands:

  • git init - Initialize a new Git repository.
  • git clone [url] - Clones the repository into the desired directory.
  • git add [file] - Stages a specific file for the next commit.
  • git add . - Stages the whole folder for the next commit
  • git commit -m "description" - Commits staged changes with user description or message
  • git push - Pushes changes to a remote repository

Branching

Branches allow you to develop features, fix bugs, or safely experiment with new ideas in a contained area of your repository. You always create a branch from an existing branch. Typically, you might create a new branch from the default branch of your repository.

  • git branch [branch-name] - Creates a new branch.
  • git checkout [branch-name] - Switches to the specified branch.
  • git merge [branch-name] - Merges changes from the selected brancg into the current branch.
